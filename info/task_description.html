The 'nonogaram' is an interesting grid puzzle.

We only use that one line in this mission.

You are given a row string (String) and clue numbers (list of Integer). Each clue numbers represents the length of continuing filled cells. They will appear in that order.

If you find more filling cells and empty cells, reflect them all in the input string and return the string. If the row string and clue numbers don't match, return None.

Example:

nonogram_row('??????????', [8]) == '??OOOOOO??'
nonogram_row('??????????', [4, 3]) == '??OO???O??'
nonogram_row('???O????O?', [3, 1]) == 'X??O??XXOX'
nonogram_row('????X?X???', [3, 2]) == '?OO?XXX?O?'
nonogram_row('O?X?O?????', [1, 3]) == 'OXX?OO?XXX'
nonogram_row('??OO?OO???O?O??', [5, 2, 2]) == 'XXOOOOOXXOOXOOX'
nonogram_row('????OO????', [4]) == 'XX??OO??XX'
nonogram_row('???X?', [0]) == 'XXXXX'
nonogram_row('?????', []) == 'XXXXX'
nonogram_row('??X??', [3]) is None
  • Symbol details:
    • '?' : unknown cell.
    • 'O' : filled cell.
    • 'X' : empty cell.

Input: two arguments, a input row (String), clue numbers(list of Integer)

Output: a result row (String) or None

How it is used:
You will get the basis of nonogram-solver.

Precondition:

  • all(r in ('?', 'O', 'X') for r in row_string)
  • all(0 ≤ cn ≤ len(row_string) for cn in clue_numbers)
  • sum(clue_numbers) + len(clue_numbers) - 1 ≤ len(row_string)
